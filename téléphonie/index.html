Module Théorique

Les fondamentaux de la VoIP pour comprendre vos manipulations.

Protocole SIP (Signaling)

Le Session Initiation Protocol est utilisé pour établir, modifier et terminer des sessions multimédias. Dans votre TP, il a permis l'enregistrement (REGISTER) de vos terminaux (Linphone, Deskphone) sur le serveur XiVO via l'adresse IP 10.172.20.10.

RTP & Codecs (Media)

Une fois la session établie par SIP, le flux audio est transporté par RTP. Vous avez utilisé le codec G.711, un codec non compressé qui offre une excellente qualité vocale mais consomme plus de bande passante que des alternatives comme G.729.

Trunking SIP & Routage

Le Trunk est un tunnel logique entre votre IPBX et un opérateur ou un autre binôme. Le routage nécessite de configurer des Inbound Routes (appels entrants) et des Outbound Routes (appels sortants via une règle de numérotation).

Échec & Limites

  • Appels externes impossibles : L'appel vers le réseau public (PSTN) a échoué.

Analyse Technique (Pourquoi ?)

L'impossibilité d'appeler l'extérieur ne provient pas d'une erreur de configuration XiVO, mais de l'absence d'un abonnement actif auprès d'un fournisseur SIP. Le Trunk est configuré, mais il n'y a pas de "passerelle" (Gateway) capable de router les appels vers le réseau téléphonique public mondial. Solution : Souscrire à un service SIP Trunk et configurer l'authentification avec l'opérateur.

Troubleshooting Guide

Répertoire des pannes classiques rencontrées.

Symptôme Cause Possible Solution
Échec d'enregistrement (Linphone/Deskphone) Identifiants erronés ou IP Serveur incorrecte. Vérifier l'ID, le mot de passe et l'IP 10.172.20.10 dans le client SIP.
Pas de son lors d'un appel (One-way audio) Mismatch de Codec ou blocage port UDP (RTP). S'assurer que G.711 est activé sur tous les terminaux et ouvert sur le firewall.
L'appel externe coupe immédiatement Pas d'abonnement ou Trunk non enregistré. Vérifier le statut du Trunk dans la CLI XiVO.
